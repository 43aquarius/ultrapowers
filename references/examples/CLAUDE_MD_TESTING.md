## 要测试的文档变体

### NULL（基线——无技能文档）
CLAUDE.md 中完全不提及技能。

### 变体 A：温和建议
```markdown
## 技能库

你可以访问 `~/.claude/skills/` 中的技能。在处理任务前
考虑检查相关技能。
```

```markdown

在处理任何任务之前，检查 `~/.claude/skills/` 中的

浏览：`ls ~/.claude/skills/`
搜索：`grep -r "keyword" ~/.claude/skills/`
```

## 测试协议

对于每个变体：

1. **先运行 NULL 基线**（无技能文档）
   - 记录代理选择哪个选项
   - 捕获确切的推理过程

2. **使用相同场景运行变体**
   - 代理是否检查技能？
   - 如果找到技能，代理是否使用？
   - 如果被违反，捕获推理过程

3. **压力测试**——添加时间/沉没成本/权威
   - 代理在压力下是否仍然检查？
   - 记录合规性何时崩溃

4. **元测试**——询问代理如何改进文档
   - "你有文档但没有检查。为什么？"
   - "文档怎样才能更清晰？"

## 成功标准

**变体成功条件：**
- 代理主动检查技能
- 代理在操作前完整阅读技能
- 代理在压力下遵循技能指导
- 代理无法绕过合规性

**变体失败条件：**
- 代理即使在无压力时也跳过检查
- 代理在未阅读的情况下"改编概念"
- 代理在压力下绕过合规性
- 代理将技能视为参考而非要求

## 预期结果

**NULL：** 代理选择最快路径，没有技能意识

**变体 A：** 代理在无压力时可能检查，在压力下跳过

**变体 B：** 代理有时检查，容易被绕过

**变体 C：** 强合规性，但可能感觉过于僵化

**变体 D：** 平衡，但较长——代理能内化吗？
