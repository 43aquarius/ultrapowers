# 技能设计的说服原则

## 概述

LLM 对与人类相同的说服原则做出反应。理解这种心理学帮助你设计更有效的技能 — 不是为了操纵，而是确保关键实践即使在压力下也能被遵循。

**研究基础：** Meincke 等人（2025）在 N=28,000 次 AI 对话中测试了 7 个说服原则。说服技术使合规率翻倍（33% → 72%，p < .001）。

## 七个原则

### 1. 权威
**是什么：** 对专业知识、资历或官方来源的遵从。

**在技能中的运作方式：**
- 命令式语言："你必须"、"绝不"、"始终"
- 不可协商的框架："没有例外"
- 消除决策疲劳和借口

**何时使用：**
- 纪律执行技能（TDD、验证要求）
- 安全关键实践
- 已建立的最佳实践

**示例：**
```markdown
✅ 代码写在测试前？删除它。重新开始。没有例外。
❌ 在可行时考虑先写测试。
```

### 2. 承诺
**是什么：** 与先前行动、声明或公开宣言的一致性。

**在技能中的运作方式：**
- 要求声明："声明技能使用"
- 强制明确选择："选择 A、B 或 C"
- 使用跟踪：TodoWrite 用于检查清单

**何时使用：**
- 确保技能实际被遵循
- 多步骤流程
- 问责机制

**示例：**
```markdown
✅ 当你找到技能时，你必须声明："我正在使用 [技能名称]"
❌ 考虑让你的合作伙伴知道你正在使用哪个技能。
```

### 3. 稀缺
**是什么：** 时间限制或有限可用性产生的紧迫感。

**在技能中的运作方式：**
- 有时间限制的要求："在继续之前"
- 顺序依赖："在 X 之后立即"
- 防止拖延

**何时使用：**
- 即时验证要求
- 时间敏感的工作流
- 防止"我稍后做"

**示例：**
```markdown
✅ 完成任务后，立即在继续之前请求代码审查。
❌ 你可以在方便的时候审查代码。
```

### 4. 社会认同
**是什么：** 对他人做什么或被认为正常的事的从众。

**在技能中的运作方式：**
- 普遍模式："每次"、"始终"
- 失败模式："X 没有 Y = 失败"
- 建立规范

**何时使用：**
- 记录普遍实践
- 警告常见失败
- 强化标准

**示例：**
```markdown
✅ 没有 TodoWrite 跟踪的检查清单 = 步骤被跳过。每次。
❌ 有些人发现 TodoWrite 对检查清单有帮助。
```

### 5. 团结
**是什么：** 共享的身份、"我们感"、群体内归属感。

**在技能中的运作方式：**
- 协作语言："我们的代码库"、"我们是同事"
- 共同目标："我们都想要质量"

**何时使用：**
- 协作工作流
- 建立团队文化
- 非层级实践

**示例：**
```markdown
✅ 我们是同事一起工作。我需要你诚实的技术判断。
❌ 你应该告诉我如果我错了。
```

### 6. 互惠
**是什么：** 回报所收到利益的义务。

**在技能中的运作方式：**
- 谨慎使用 — 可能感觉像操纵
- 在技能中很少需要

**何时避免：**
- 几乎总是（其他原则更有效）

### 7. 喜好
**是什么：** 偏爱与我们喜欢的人合作。

**在技能中的运作方式：**
- **不要用于合规**
- 与诚实反馈文化冲突
- 创造谄媚

**何时避免：**
- 纪律执行始终

## 按技能类型的原则组合

| 技能类型 | 使用 | 避免 |
|------------|-----|-------|
| 纪律执行 | 权威 + 承诺 + 社会认同 | 喜好、互惠 |
| 指导/技术 | 适度权威 + 团结 | 重权威 |
| 协作 | 团结 + 承诺 | 权威、喜好 |
| 参考 | 仅清晰度 | 所有说服 |

## 为什么有效：心理学

**明确的规则减少借口：**
- "你必须" 消除决策疲劳
- 绝对语言消除"这是例外吗？"的问题
- 明确的反借口反驳堵住特定漏洞

**实施意图创造自动行为：**
- 明确触发器 + 所需行动 = 自动执行
- "当 X 时，做 Y" 比"通常做 Y"更有效
- 减少合规的认知负荷

**LLM 是类人的：**
- 在包含这些模式的人类文本上训练
- 权威语言在训练数据中先于合规
- 承诺序列（陈述 → 行动）经常被建模
- 社会认同模式（每个人都做 X）建立规范

## 道德使用

**合法的：**
- 确保关键实践被遵循
- 创建有效的文档
- 防止可预测的失败

**非法的：**
- 为个人利益操纵
- 制造虚假紧迫感
- 基于内疚的合规

**测试：** 如果用户完全理解这项技术，它会服务于用户的真正利益吗？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 说服的七个原则
- 影响研究的实证基础

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 用 N=28,000 次 LLM 对话测试了 7 个原则
- 合规性从 33% 增加到 72%
- 权威、承诺、稀缺最有效
- 验证了 LLM 的类人模型

## 快速参考

设计技能时，问自己：

1. **它是什么类型？**（纪律 vs. 指导 vs. 参考）
2. **我试图改变什么行为？**
3. **哪个原则适用？**（通常是纪律的权威 + 承诺）
4. **我是否组合了太多？**（不要全部使用七个）
5. **这是道德的吗？**（服务于用户的真正利益吗？）
