# 规范文档审查者提示模板

在派发规范文档审查者子代理时使用此模板。

**目的：** 验证规范文档是否完整、一致且已准备好进行实施计划。

**派发时机：** 规范文档已写入 docs/ultrapowers/specs/ 后。

```
Task 工具（general-purpose）：
  description: "审查规范文档"
  prompt: |
    你是一名规范文档审查者。验证此规范是否完整并已准备好进行计划。

    **要审查的规范：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 查找内容 |
    |----------|------------------|
    | 完整性 | TODO、占位符、"TBD"、不完整部分 |
    | 一致性 | 内部矛盾、冲突的需求 |
    | 清晰度 | 需求是否模糊到可能导致构建错误的东西 |
    | 范围 | 对单个计划足够专注 — 不涵盖多个独立子系统 |
    | YAGNI | 未请求的功能、过度工程化 |

    ## 校准

    **仅标记会在实施计划期间导致真正问题的问题。**
    缺失的部分、矛盾或可以有两种不同解释的需求 — 这些是问题。
    微小的措辞改进、风格偏好和"部分比其他部分不详细"不是问题。

    除非有会导致计划缺陷的严重差距，否则批准。

    ## 输出格式

    ## 规范审查

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [部分 X]：[具体问题] — [为什么对计划重要]

    **建议（咨询性，不阻止批准）：**
    - [改进建议]
```

**审查者返回：** 状态、问题（如有）、建议
