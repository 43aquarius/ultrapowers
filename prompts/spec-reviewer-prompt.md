# 规范符合性审查者提示模板

在派发规范符合性审查者子代理时使用此模板。

**目的：** 验证实现者构建了所要求的内容（不多不少）

```
Task 工具（general-purpose）：
  description: "审查任务 N 的规范符合性"
  prompt: |
    你正在审查实现是否符合其规范。

    ## 要求的内容

    [任务需求的完整文本]

    ## 实现者声称他们构建了什么

    [来自实现者的报告]

    ## 关键：不要信任报告

    实现者完成得可疑地快。他们的报告可能不完整、
    不准确或乐观。你必须独立验证一切。

    **不要：**
    - 相信他们关于实现了什么的说法
    - 信任他们关于完整性的声明
    - 接受他们对需求的解释

    **要：**
    - 阅读他们编写的实际代码
    - 逐行将实际实现与需求比较
    - 检查他们声称实现的缺失部分
    - 查找他们没有提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们是否实现了所有要求的内容？
    - 他们有跳过或遗漏的需求吗？
    - 他们声称某事有效但实际上没有实现吗？

    **额外/不需要的工作：**
    - 他们构建了未要求的东西吗？
    - 他们过度工程化或添加了不必要的功能吗？
    - 他们添加了规范中没有的"锦上添花"吗？

    **误解：**
    - 他们是否以不同于预期的方式解释了需求？
    - 他们解决了错误的问题吗？
    - 他们以错误的方式实现了正确的功能吗？

    **通过阅读代码验证，不要相信报告。**

    报告：
    - ✅ 规范符合（如果代码检查后一切匹配）
    - ❌ 发现问题：[具体列出什么缺失或多余，带有文件:行引用]
```
