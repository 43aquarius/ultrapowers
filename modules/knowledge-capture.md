
# 持续学习

## 概述

从会话中提取可重用模式，存储为带有置信度评分的原子"本能"。本能演化为集群，然后演化为完整技能、命令或代理。

**核心原则：** 从每次会话中学习。原子观察 → 集群 → 可重用技能。

**为什么限定项目范围：** React 模式不应泄漏到 Python 项目。领域特定知识留在其所属位置。

## 何时使用

**在以下情况使用：**
- 会话结束时（通过停止钩子或手动触发）
- 发现非显而易见的解决方案时
- 解决棘手 bug 时
- 用户纠正你的方法时
- 发现项目特定模式时

**在以下情况不使用：**
- 标准实践（已有文档）
- 无重用价值的一次性修复
- CLAUDE.md 或技能中已有的信息

## 学习管道

```
会话 → 本能 → 集群 → 技能/命令/代理
      （原子）  （分组）   （晋升）
```

### 第 1 层：本能（原子观察）

每个本能是一个单一的原子观察：

```markdown
---
name: python-async-timeout-pattern
confidence: 0.7
domain: python
project: my-api-service
source: session-2025-01-15
tags: [async, timeout, error-handling]
---

## 观察
当对数据库连接使用 `asyncio.wait_for()` 时，
对于跨多个查询的事务，始终设置 `timeout=None`。
默认 30 秒超时会在负载下导致静默失败。

## 背景
在调试用户服务中间歇性 500 错误时发现。
连接池耗尽，因为超时的协程没有释放连接。

## 操作
对于多查询事务，使用：
```python
async with db.transaction(timeout=None):
    ...
```
```

**置信度评分：**
- 0.3 — 单次观察，未验证
- 0.5 — 看到两次或用户确认
- 0.7 — 在同一项目的多个会话中看到
- 0.9 — 在多个项目中确认

### 第 2 层：集群（相关本能）

当 3+ 个本能共享一个领域和主题时，它们形成一个集群：

```markdown
集群：Python 异步模式
本能：
  - python-async-timeout-pattern (0.7)
  - python-async-connection-pool (0.8)
  - python-async-retry-backoff (0.6)
平均置信度：0.7
```

### 第 3 层：晋升（集群 → 技能）

晋升标准：
- **2+ 个项目**中出现相同本能，平均置信度 ≥ 0.8
- 集群有 **5+ 个本能**，平均置信度 ≥ 0.7
- 用户明确要求晋升

**晋升目标：**
| 目标 | 时机 |
|------|------|
| 技能 | 通用技术，广泛适用 |
| 命令 | 常用工作流，值得一个斜杠命令 |
| 代理 | 复杂的多步骤流程，受益于专业化 |
| CLAUDE.md 条目 | 项目特定约定 |

## 存储

```
~/.claude/memory/instincts/
  project-name/
    instinct-name.md
  clusters/
    cluster-name.md
  promoted/
    skill-name/
      SKILL.md
```

**项目范围的本能**放在 `project-name/` 中。它们不会被其他项目加载。

**跨项目本能**（置信度 ≥ 0.9，在 3+ 个项目中看到）放在根目录，随处可用。

## 基于钩子的捕获

技能对捕获不可靠（50-80% 触发率）。使用钩子实现 100% 可靠性：

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "*",
        "hooks": [{
          "type": "command",
          "command": "node ~/.claude/scripts/hooks/extract-instincts.js"
        }]
      }
    ]
  }
}
```

钩子：
1. 读取会话记录
2. 识别模式（解决的错误、用户纠正、变通方案）
3. 生成本能草稿
4. 以 `confidence: 0.3`（单次观察）存储

## 手动提取

当钩子不可用时，在会话结束时手动提取：

1. **回顾会话：** 我学到了什么不明显的东西？
2. **检查纠正：** 用户是否纠正了我的方法？
3. **检查变通方案：** 我是否找到了非标准解决方案？
4. **检查错误：** 我是否遇到了需要调查的错误？
5. **编写本能：** 每个观察一个，原子化，带背景。

## 集成

**相关技能：**
- **ultrapowers:skill-authoring** — 用于将集群晋升为完整技能
- **ultrapowers:root-cause-analysis** — Bug 解决是高价值本能来源
- **ultrapowers:red-green-cycle** — 测试模式是可重用本能

**相关 ECC 技能：**
- `continuous-learning-v2` — 带有演化管道的高级版本
- `knowledge-ops` — 多层知识管理
- `iterative-retrieval` — 在创建新本能前查找现有本能

## 危险信号

- 为标准实践创建本能（噪声，不是信号）
- 跨项目污染（React 模式出现在 Python 项目中）
- 单次观察的高置信度（保持 0.3 直到验证）
- 从不晋升集群（本能应演化为技能）
- 在全局范围存储而项目范围更合适

## 底线

**每次会话都产生知识。原子化捕捉，自信评分，适当范围，证明后晋升。**

本能 → 集群 → 技能。这就是框架从使用中成长的方式。
